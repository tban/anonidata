# Building AnoniData for Distribution

## Prerequisites
1. Ensure you have the development environment set up (Node.js, Rust 1.80+, and Python environment).
2. Have code signing certificates configured if building for macOS (configured in `src-tauri/tauri.conf.json`).

## Build Steps

### 1. Build the Python Backend
If you make changes to the Python code, rebuild the backend binary:
```bash
npm run build:backend
```
This runs PyInstaller and outputs the compiled binary to `dist/anonidata-backend`.

### 2. Build the Tauri Application
To compile the frontend, compile the Rust backend, bundle the Python sidecar, and produce the final installer:
```bash
npm run build
```
This triggers the Tauri CLI:
- Builds the Vite frontend to `dist/renderer/`.
- Packages the Python sidecar binary from `src-tauri/binaries/` (resolving the symlink to `dist/anonidata-backend`).
- Compiles the Rust wrapper and outputs native installation packages (DMG on macOS, NSIS EXE on Windows).

### 3. Output Location
Distribution packages are generated by Tauri in the following target directories:
- **macOS DMG**: `src-tauri/target/release/bundle/dmg/AnoniData_<version>_aarch64.dmg` (for Apple Silicon)
- **Windows EXE**: `src-tauri/target/release/bundle/nsis/AnoniData_<version>_x64-setup.exe`

## Important Configuration Notes

### Backend Binary Path (CRITICAL)
Tauri v2 requires the sidecar binary to have the host's target triple appended. For example, on Apple Silicon macOS, Tauri expects:
`src-tauri/binaries/anonidata-backend-aarch64-apple-darwin`

This is set up as a symlink pointing to the compiled PyInstaller binary:
`ln -sf ../../dist/anonidata-backend src-tauri/binaries/anonidata-backend-aarch64-apple-darwin`

The binary configuration is registered in `src-tauri/tauri.conf.json` under:
```json
"externalBin": [
  "binaries/anonidata-backend"
]
```

### Entitlements File
The file `build/entitlements.mac.plist` must exist for code signing:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>
    <key>com.apple.security.cs.disable-library-validation</key>
    <true/>
    <key>com.apple.security.automation.apple-events</key>
    <true/>
</dict>
</plist>
```

## Clean Build
To do a clean build:
```bash
rm -rf src-tauri/target
npm run build
```

## Troubleshooting

### "Sidecar not found" or launch issues
If you see launch issues:
1. Verify that `dist/anonidata-backend` exists.
2. Verify that the symlink `src-tauri/binaries/anonidata-backend-aarch64-apple-darwin` (or target triple matching your OS) exists and points to `../../dist/anonidata-backend`.
3. Check `src-tauri/capabilities/default.json` to ensure permissions allow executing the sidecar binary.
